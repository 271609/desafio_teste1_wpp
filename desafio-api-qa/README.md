README – Testes de API (Desafio Técnico)

Este repositório contém a solução desenvolvida para o Desafio de Testes de API, utilizando a API pública JSONPlaceholder.
A proposta inclui criação de cenários BDD, automação de testes via Postman, análise crítica da API e documentação completa.

1.Tecnologias Utilizadas
- Postman
-Utilizado como ferramenta principal para construção e execução dos testes de API.
-Permite organizar cenários em collections estruturadas

 
2.Como Configurar e Executar os Testes
-Requisitos: 
-Postman Desktop instalado ou
-Extensão Postman Web + Postman Agent

-Importar Collection no Postman ;
https://cintiadb.postman.co/workspace/My-Workspace~772682b9-5d57-42fa-8481-a87453b28569/collection/17010986-45a9eeee-a1a2-4f77-9cbc-98e3ba8b63ee?action=share&source=copy-link&creator=17010986


Executar via Postman
🔹 Rodar um endpoint individual
Clique no endpoint
Clique em Send
🔹 Rodar a collection inteira
Vá até a collection
Clique em Run Collection
Pressione Start Run
O Postman exibirá:
Status de cada requisição
Scripts de teste
Indicadores de sucesso/falha
